RMA Kinetics is a library of kinetic models for synthetic serum reporters called Released Markers of Activity (RMAs). The core library is written in Rust for performance, with Python bindings for ergonomic integration with scientific computing tools such as NumPy. Both APIs expose the same underlying models and solvers.

## What are RMAs?

Released Markers of Activity (RMAs) are synthetic serum reporters engineered into specific cell populations. When those cells are active, they secrete a detectable protein into the bloodstream. Measuring RMA concentration in a blood sample lets researchers infer underlying neural or cellular activity without invasive procedures.

RMA Kinetics models the full pharmacokinetic trajectory of these reporters: from gene expression in the brain, through the blood-brain barrier, and into the plasma. Models also incorporate doxycycline-controlled gene silencing, chemogenetic (DREADD) induction, and oscillating expression dynamics.

## Available models

Each model lives in a corresponding submodule under `rma_kinetics.models` (Python) or `rma_kinetics::models` (Rust). Every submodule exposes at least a `Model` struct/class and a `State` struct/class.

## Rust and Python APIs

The core library is published on [crates.io](https://crates.io/crates/rma-kinetics) as `rma_kinetics`. You can add it to any Rust project with a single command:

```bash
cargo add rma_kinetics
```

The Python bindings are built from source using [maturin](https://www.maturin.rs/) and expose the same models and solvers through a Pythonic interface. See the [quickstart](/quickstart) for step-by-step installation instructions.

<Note>
  The Python package on PyPI uses an older pure-Python implementation. For the latest Rust-backed bindings, build from source using maturin as described in the [quickstart](/quickstart).
</Note>
