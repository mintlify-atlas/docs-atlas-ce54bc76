---
title: "Get started with RMA Kinetics"
description: "Install the RMA Kinetics library and run your first kinetic simulation using the constitutive model with a Dormand-Prince solver in Python or Rust."
---

RMA Kinetics is available as a Rust crate on crates.io and as a Python package built via maturin. Follow the steps below to install the library, create a model, and run a simulation over a time span of your choice.

<Steps>
  <Step title="Install the library">
    Choose your language to get the installation command.

    <Tabs>
      <Tab title="Python">
        The Python bindings are built from source using [maturin](https://www.maturin.rs/). Clone the repository, create a virtual environment, and build the package in development mode:

        ```bash
        git clone https://github.com/nsbuitrago/rma-kinetics-rs
        cd rma-kinetics-rs
        uv venv
        uvx maturin develop --uv --features py -m crates/rma-kinetics/Cargo.toml
        ```

        <Note>
          The Python package on PyPI (`rma-kinetics`) uses an older pure-Python implementation. The commands above build the current Rust-backed bindings locally. A Rust toolchain must be installed before running `maturin develop`. See the [Rust installation guide](https://www.rust-lang.org/tools/install) if needed.
        </Note>
      </Tab>
      <Tab title="Rust">
        The `rma_kinetics` crate is published on [crates.io](https://crates.io/crates/rma-kinetics). Add it to your project with cargo:

        ```bash
        cargo add rma_kinetics
        ```

        Or add it manually to your `Cargo.toml`:

        ```toml
        [dependencies]
        rma_kinetics = "0.1.1"
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Import the model and solver">
    Each model lives in its own submodule. For a constitutive model, import `Model` and `State` from the `constitutive` submodule, and choose a solver.

    <CodeGroup>

    ```python python
    from rma_kinetics.models.constitutive import Model, State
    from rma_kinetics.solvers import Dopri5
    ```

    ```rust rust
    use rma_kinetics::models::constitutive::{Model, State};
    use rma_kinetics::Solve;
    use differential_equations::methods::ExplicitRungeKutta;
    ```

    </CodeGroup>

    <Tip>
      `Dopri5` in Python and `ExplicitRungeKutta::dopri5()` in Rust both use the Dormand-Prince 5(4) explicit Runge-Kutta method, which is a good general-purpose adaptive solver for these ODE systems.
    </Tip>
  </Step>

  <Step title="Create a model and initial state">
    Construct a `Model` and an initial `State`. Both use sensible defaults — the default constitutive model parameters reflect RMA expression in the CA1 region of the hippocampus under a human-synapsin promoter, and the default initial state sets both `brain_rma` and `plasma_rma` to zero.

    <CodeGroup>

    ```python python
    model = Model()
    # Or with explicit parameters:
    # model = Model(prod=0.2, bbb_transport=0.6, deg=0.007)

    init_state = State()
    # Or with non-zero initial conditions:
    # init_state = State(brain_rma=0.0, plasma_rma=0.0)

    solver = Dopri5()
    ```

    ```rust rust
    let model = Model::default();
    let init_state = State::zeros();
    let mut solver = ExplicitRungeKutta::dopri5();
    ```

    </CodeGroup>
  </Step>

  <Step title="Run the simulation">
    Call `solve` with a start time `t0`, end time `tf`, step size `dt`, the initial state, and the solver. The method returns a `Solution` object containing the time series for all species.

    <CodeGroup>

    ```python python
    t0 = 0
    tf = 100
    dt = 1

    solution = model.solve(t0, tf, dt, init_state, solver)
    ```

    ```rust rust
    let t0 = 0.;
    let tf = 100.;
    let dt = 1.;

    let solution = model.solve(t0, tf, dt, init_state, &mut solver);
    assert!(solution.is_ok());
    let solution = solution.unwrap();
    ```

    </CodeGroup>
  </Step>

  <Step title="Access the results">
    Extract concentration time series from the returned `Solution`. In Python, `Solution` fields are NumPy arrays. In Rust, `solution.y` is a `Vec` of `State` structs that you map over to extract individual fields.

    <CodeGroup>

    ```python python
    plasma_rma = solution.plasma_rma  # numpy array of plasma RMA over time
    brain_rma = solution.brain_rma    # numpy array of brain RMA over time
    ts = solution.ts                  # numpy array of time points

    print(f"Final plasma RMA: {plasma_rma[-1]:.4f}")
    print(f"Final brain RMA:  {brain_rma[-1]:.4f}")
    ```

    ```rust rust
    let plasma_rma: Vec<f64> = solution.y
        .iter()
        .map(|state| state.plasma_rma)
        .collect();

    let brain_rma: Vec<f64> = solution.y
        .iter()
        .map(|state| state.brain_rma)
        .collect();

    let ts = &solution.t;

    println!("Final plasma RMA: {:.4}", plasma_rma.last().unwrap());
    println!("Final brain RMA:  {:.4}", brain_rma.last().unwrap());
    ```

    </CodeGroup>
  </Step>
</Steps>
